CustomPlaygroundDisplayConvertible

A type that supplies a custom description for playground logging.
プレイグラウンドログのためにあつらえの記述を提供する型。

Overview
概要

All types have a default description for playgrounds. This protocol allows types to provide custom descriptions which are then logged in place of the original instance.
全ての型は、プレイグラウンドに対する省略時の記述を持ちます。このプロトコルは、型にあつらえの記述を提供させます、それは元々のインスタンスの代わりに記録されます。

Playground logging can generate, at a minimum, a structured description of any type. Playground logging is also capable of generating a richer, more specialized description of core types – for instance, the contents of a String are logged, as are the components of an NSColor or UIColor.
プレイグラウンドログは、最低でも、あらゆる型の郷増加された記述を生成できます。プレイグラウンドログはまた、より豊かな、もっと特殊化された中核の型の記述を生成する能力も持ちます – 例えば、Stringの内容は、NSColorまたはUIColorの構成要素であるように記録されます。

The current playground logging implementation logs specialized descriptions of at least the following types:
現在のプレイグラウンドログ実装は、少なくとも以下の型の特殊化された記述を記録します：

  • String and NSString

  • Int and UInt (including the sized variants)

  • Float and Double

  • Bool

  • Date and NSDate

  • NSAttributedString

  • NSNumber

  • NSRange

  • URL and NSURL

  • CGPoint, CGSize, and CGRect

  • NSColor, UIColor, CGColor, and CIColor

  • NSImage, UIImage, CGImage, and CIImage

  • NSBezierPath and UIBezierPath

  • NSView and UIView

Playground logging may also be able to support specialized descriptions of other types.
プレイグラウンドログはまた、他の型の特殊化された記述のサポートが可能です。

Implementors of CustomPlaygroundDisplayConvertible may return a value of one of the above types to also receive a specialized log description. Implementors may also return any other type, and playground logging will generated structured logging for the returned value.
CustomPlaygroundDisplayConvertibleの実装は、上の型の１つの値を返して、また特殊化されたログ記述を受け取るかもしれません。実装側はまた、何らかの他の型を返すかもしれません、そしてプレイグラウンドログは構造化されたログを戻り値に対して生成するでしょう。

Note
注意

CustomPlaygroundDisplayConvertible conformances chain – that is, if playgroundDescription returns an instance which itself conforms to CustomPlaygroundDisplayConvertible, then playground logging will ask for that instance’s playgroundDescription and so on. It is permissible for playground logging implementations to place a reasonable limit on this kind of chaining to prevent infinite loops.
CustomPlaygroundDisplayConvertible準拠は連鎖します - すなわち、playgroundDescriptionがそれ自身CustomPlaygroundDisplayConvertibleに準拠するインスタンスを返すならば、そのときプレイグラウンドログはそのインスタンスのもつplaygroundDescriptionについて尋ねるでしょう、そしてまた。この種の連鎖に賢明な制限を置いて無限ループを防ぐことは、プレイグラウンドログ実装に差し支えありません。
