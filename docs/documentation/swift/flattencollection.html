FlattenCollection

A flattened view of a base collection of collections.
複数コレクションからなる基盤コレクションを平坦に見たもの。

Overview
概要

The elements of this view are a concatenation of the elements of each collection in the base.
この見方での要素たちは、その基盤の中の各コレクションの要素の連結です。

The joined method is always lazy, but does not implicitly confer laziness on algorithms applied to its result. In other words, for ordinary collections c:
joinedメソッドは常に遅延です、しかしそれの結果に適用されるアルゴリズム上に、遅延性を暗黙的に授けられません。言い換えれば、よくあるコレクションcに対して：

  • c.joined() does not create new storage
    c.joined()は新しいストレージを作りません

  • c.joined().map(f) maps eagerly and returns a new array
    c.joined().map(f)は先行にマップを行い、新しい配列を返します

  • c.lazy.joined().map(f) maps lazily and returns a LazyMapCollection
    c.lazy.joined().map(f)は遅延にマップして、LazyMapCollectionを返します

Note
注意

The performance of accessing startIndex, first, any methods that depend on startIndex, or of advancing a FlattenCollectionIndex depends on how many empty subcollections are found in the base collection, and may not offer the usual performance given by Collection or ForwardIndex. Be aware, therefore, that general operations on FlattenCollection instances may not have the documented complexity.
startIndex、first、あらゆるstartIndex依存のメソッドでのアクセスにおける、またはFlattenCollectionIndexでの進んでいくことにおける性能は、どのくらい多くの空の下位コレクションが基盤コレクションにおいて見つけられるかに依存します、そしてCollectionまたはForwardIndexによって与えられる通常の性能を呈さないでしょう。自覚すべきは、したがって、FlattenCollectionインスタンス上での操作全般が複雑に記述されるものを持たないようにすることです。

  • See also: FlattenSequence
    次を参照してください：FlattenSequence
