NSProgress

Declaration 宣言

Overview 概要

The NSProgress class provides a self-contained mechanism for progress reporting. It makes it easy for code that does work to report the progress of that work, and for user interface code to observe that progress for presentation to the user. Specifically, it can be used to show the user a progress bar and explanatory text, both updated properly as progress is made. It also allows work to be cancelled or paused by the user. NSProgressクラスは、プログレスリポート（進捗報告）のための自己充足機構を提供します。それは、作業をするコードに対してその作業の進捗を報告しやすく、そしてユーザインターフェイスコードに対してユーザに提示する進捗を監視しやすくします。とりわけ、それは進捗バーと説明テキストを、両方とも進捗が進むにつれて適切に更新された状態で、ユーザに表示するために使われます。それはまた作業に、ユーザによって取り消されるまたは一時停止されることを可能にします。

Reporting Progress 進捗を報告する

Using the methods of this class, your code can report the progress it’s currently making toward completing some task, including progress in related subtasks. このクラスのメソッドを使って、あなたのコードは、あるタスクの完了に向かって進んでいる現在の進捗を、関連する下位タスクの進捗を含めて報告できます。 You can create instances of this class using the initWithParent:userInfo: instance method or the progressWithTotalUnitCount: class method.

Progress objects have a number of properties that you can use to observe and report current progress. For instance, the totalUnitCount property represents the total number of units of work that need to be performed, and the completedUnitCount and fractionCompleted properties represents how much of that work has already been completed. The fractionCompleted property is useful for updating progress indicators or textual descriptors; to check whether progress is complete, you should test that completedUnitCount >= totalUnitCount (assuming, of course, that totalUnitCount > 0). 進捗オブジェクトは、幾つかのプロパティを持ちます、それはあなたが現在の進捗を監視および報告するのに使用できるものです。たとえば、totalUnitCountプロパティは遂行されるのに必要とされる仕事の単位の総数を表します、そしてcompletedUnitCountとfractionCompletedプロパティはどのくらいその仕事が既に完了したかを表します。fractionCompletedプロパティは、進捗インジケータまたはテキスト記述子を更新するのに役立ちます；進捗が完了するかどうか調べるには、あなたはcompletedUnitCount >= totalUnitCountをテストするべきです（もちろんtotalUnitCount > 0と仮定して）。

The following listing shows a sample method that reports the progress of performing some operation on a piece of data. When the progress object is first created, the value of its totalUnitCount property is set to some suitable batch size for this operation, and the completedUnitCount count is 0. Each time the loop executes and the batch of data is processed, the progress object’s completedUnitCount property is incremented appropriately. 以下のコード出力は、ある見本のメソッドを示します、それはある演算を一片のデータ上で実行することの進捗を報告するものです。進捗オブジェクトが最初に作成される時、それのtotalUnitCountプロパティの値はこの演算に対するある適切な一括処理寸法（バッチサイズ）に設定されます、そしてcompletedUnitCount計数は0です。ループの遂行と一括データが処理されるたびごとに、進捗オブジェクトのもつcompletedUnitCountプロパティは適切に漸増されます。

Each of the properties of a progress object, including totalUnitCount, completedUnitCount, and fractionCompleted, support Key-Value Observing. This makes it extremely easy for a view or window controller object to observe the properties, and update UI elements such as progress indicators when the values change. It also means that there is a non-zero cost to updating the values of these properties, so you should avoid using a unit count that is too granular—if you’re iterating over a large data set, for example, and each operation takes only a trivial amount of time, you should divide the work into batches so you can update the unit count once per batch rather than once per iteration. ある進捗オブジェクトのプロパティのそれぞれは、totalUnitCount、completedUnitCount、そしてfractionCompletedを含めて、キー値監視をサポートします。これは、あるビューまたはウインドウのコントローラオブジェクトが、それらプロパティを監視する、そして値が変化するにつれて進捗インジケータのようなUI要素を更新する、のを非常に簡単にします。それはまた、これらプロパティの値を更新するための皆無ではないコストがあることを意味します、それであなたは微粒すぎる単位計数を使うことを避けるべきです — あなたがある大きなデータ群すべてにわたって反復している、例えば、そして各演算は取るに足らない量の時間しかかからないならば、あなたはその仕事をいくつかの一括処理に分割すべきです、それであなたは一括処理ごとに一度だけ単位計数を更新できます、反復ごとに一度でなく。

Creating a Tree of Progress Objects 進捗オブジェクトのツリーを作成する

Often, your code may need to report the overall progress of an operation that is composed of several sub-operations. To accomplish this, your code can report the progress of each sub-operation by building up a tree of progress objects. しばしば、あなたのコードはある演算の全体の進捗を報告する必要があるかもしれません、それは幾つかの下位演算から構成されます。これを成し遂げるために、あなたのコードは、進捗オブジェクトいくらかからなるある木構造（ツリー）を作り上げることによって、各下位演算の進捗を報告することが可能です。

The NSProgress reporting mechanism supports a loosely coupled relationship between progress objects. Sub-operations don’t need to know anything about the parent progress item—you can simply create new progress objects that are added as a child of another NSProgress instance. The child object is assigned a portion of the parent’s pending unit count. When the children complete, the parent’s completedUnitCount property is automatically increased by a predefined amount. NSProgress報告機構は、進捗オブジェクトの間のゆるく組みにされた関係をサポートします。下位演算は親進捗項目について何も知る必要はありません — あなたは、別のNSProgressインスタンスの子として加えられる新しい進捗オブジェクトを単純に作成できます。子オブジェクトは、親のもつ未解決の単位計数の一端を割り当てられます。子が完了する場合、親のもつcompletedUnitCountプロパティはあらかじめ定義された量によって自動的に漸増されます。

You add child progress objects to your tree implicitly or explicitly. あなたは、子進捗オブジェクトをあなたのツリーに暗黙的にまたは明示的に加えます。

Implicitly Adding a Child 暗黙的に子を加える

Add a child implicitly by setting a pending unit count for the parent and creating a new NSProgress instance. That new instance will be set as a child of the parent and be assigned the pending unit count. 親に対してある未解決の単位計数を設定することそして新しいNSProgressインスタンスを作成することによって、子を暗黙的に加えてください。この新しいインスタンスは、親の子として設定されます、そして未解決の単位計数を割り当てられます。

As an example, consider that you are tracking the progress of code downloading and copying files on disk. You could use a single progress object to track the entire task, but it’s easier to manage each subtask using a separate progress object. 例として、あなたがディスク上でファイルをダウンロードおよびコピーしているコードの進捗を追跡していると考えてください。あなたは単一の進捗オブジェクトを使ってタスク全体を追跡できます、しかし独立した進捗オブジェクトを使って各下位タスクを管理する方がより簡単です。 You start by creating an overall parent progress object with a suitable total unit count, then call becomeCurrentWithPendingUnitCount:, then create your sub-task progress objects, before finally calling resignCurrent.

The pending unit count that you specify in the first method is divided equally among the child progress objects you create between these two method calls. Each child progress object maintains its own internal unit count, but when it completes its units (that is, the child object’s completedUnitCount == totalUnitCount), the parent progress object’s completedUnitCount is increased by the assigned portion of the original pending unit count. あなたが最初のメソッドにおいて指定する未解決単位計数は、これら２つのメソッド呼び出しの間にあなたが作成する子進捗オブジェクトのそれぞれに等しく分割されます。各子進捗オブジェクトはそれ自身の内部的な単位計数を保守します、しかしそれがそれの単位それらを完了する（すなわち、子オブジェクトでcompletedUnitCount == totalUnitCountである）時、親進捗オブジェクトのもつcompletedUnitCountは、元の未解決単位計数の割当分によって漸増されます。

In the following example, the overall parent progress has 100 units; the two child objects therefore get 50 pending units each, and keep track internally of 10 units of work each. When each child completes its 10 units, the parent’s completed unit count is increased by 50. 以下の例において、全体としての親進捗は100単位を持ちます；２つの子オブジェクトはそれゆえに50の未解決単位をそれぞれ受け取ります、そして内部的に10単位の仕事の経過をそれぞれ追います。それぞれの子がそれの10単位を完了する時、親のもつ完了された単位計数は、50によって漸増されます。

If you don’t create any child progress objects between the calls to becomeCurrentWithPendingUnitCount: and resignCurrent, the “parent” progress automatically updates its completedUnitCount by adding the pending units.

Explicitly Adding a Child 明示的に子を加える

As of iOS 9.0 and OS X v10.11 you can explicitly add a child to a progress tree. iOS 9.0およびOS X v10.11現在あなたは明示的に子を進捗ツリーに追加できます。

To add a child, call addChild:withPendingUnitCount: on the parent. The value for pending unit count is the amount of the parent’s totalUnitCount consumed by the child. 未解決単位計数に対する値は、親のもつtotalUnitCountのうち子によって消費される分の合計です。 The child usually follows the NSProgressReporting protocol.

In the following example, the overall parent progress has 10 units. The child progress for the download gets 8 units and tracks the download of a photo. The progress for the filter takes a lot less time and gets the remaining 2 units. When the download completes the parent’s completed unit count is updated by 8. When the filter completes it is updated by the remaining 2 units. 以下の例において、全体としての親進捗は10単位を持ちます。ダウンロードに対する子進捗は、８単位を受け取ります、そしてある写真のダウンロードを追跡します。フィルタに対する進捗は、ずっと少ない時間しかかかりません、それで残りの２単位を受け取ります。ダウンロードが完了する場合は親のもつ完了単位計数は８だけ更新されます。フィルタが完了する場合はそれは残りの２単位で更新されます。

Topics 話題

Creating Progress Objects 進捗オブジェクトを作成する

- initWithParent:userInfo:

Initializes a newly allocated NSProgress instance. 新しくアロケートされたNSProgressインスタンスを初期化します。

+ discreteProgressWithTotalUnitCount:

Creates and returns an NSProgress instance with the specified totalUnitCount that is not part of any existing progress tree. あるNSProgressインスタンスをこの指定されたtotalUnitCountで作成して返します、それはどんな既存の進捗ツリーの一部でもありません。 The instance is initialized using initWithParent:userInfo: with the parent set to nil.

+ progressWithTotalUnitCount:

Creates and returns an NSProgress instance, initialized using initWithParent:userInfo:.

+ progressWithTotalUnitCount:parent:pendingUnitCount:

Creates and returns an NSProgress instance attached to the specified parent with the totalUnitCount set to portionOfParentTotalUnitCount. 指定された親に付属したあるNSProgressインスタンスを作成して返します、totalUnitCountにportionOfParentTotalUnitCountが設定された状態です。

Current Progress Object 現在の進捗オブジェクト

+ currentProgress

Returns the NSProgress instance, if any, associated with the current thread by a previous invocation of becomeCurrentWithPendingUnitCount:.

- becomeCurrentWithPendingUnitCount:

Sets the receiver as the current progress object of the current thread and specifies the portion of work to be performed by the next child progress object of the receiver. レシーバを現在のスレッドの現在の進捗オブジェクトとして設定します、そしてこの部分の作業をレシーバの次の子進捗オブジェクトによって実行されるように指定します。

- addChild:withPendingUnitCount:

Add a process object as a child of a progress tree. The inUnitCount indicates the expected work for the progress unit. ある進捗オブジェクトを進捗ツリーの子として加えます。inUnitCountは、進捗単位に対する期待される仕事を指し示します。

- resignCurrent

Balance the most recent previous invocation of becomeCurrentWithPendingUnitCount: on the same thread by restoring the current progress object to what it was before becomeCurrentWithPendingUnitCount: was invoked.

Reporting Progress 進捗を報告する

totalUnitCount

The total number of units of work tracked for the current progress. 現在の進捗に追跡される仕事の単位の総数。

completedUnitCount

The number of units of work for the current job that have already been completed. 既に完了された、現在のジョブに対する仕事の単位の数。

localizedDescription

A localized description of progress tracked by the receiver. レシーバによって追跡される進捗のローカライズされた記述。

localizedAdditionalDescription

A more specific localized description of progress tracked by the receiver. レシーバによって追跡される進捗のより具体的なローカライズされた記述。

Observing Progress 進捗を監視する

fractionCompleted

The fraction of the overall work completed by this progress object, including work done by any children it may have. 仕事全体のうちのこの進捗オブジェクトによって完了された部分、それが持つかもしれないいくらかの子によって行われた仕事を含んでいます。

Controlling Progress 進捗を制御する

pausable

paused

Indicates whether the receiver is tracking work that has been paused. レシーバが追跡しているのは一時停止されてしまった仕事かどうかを指し示します。

- pause

Pause progress tracking. 進捗の追跡を一時停止します。

pausingHandler

- resume

resumingHandler

The block to invoke when progress is resumed. 進捗が再開される時に発動するブロック。
