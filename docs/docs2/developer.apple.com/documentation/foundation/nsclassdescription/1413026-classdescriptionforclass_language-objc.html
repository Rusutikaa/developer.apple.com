Type Method 型メソッド

classDescriptionForClass:

Returns the class description for a given class. 与えられたクラスのクラス記述を返します。

SDK

  • macOS 10.0+

Framework フレームワーク

  • Foundation ファウンデーション

On This Page この項には

  • Declaration 宣言
  • Parameters パラメータ
  • Return Value 戻り値
  • Discussion 議論
  • See Also 参照

Declaration 宣言

Parameters パラメータ

aClass

The class for which to return a class description. See note below for important details. それに対してクラス記述を返すクラス。下の注意を重要な詳細として見てください。

Return Value 戻り値

The class description for aClass, or nil if a class description cannot be found. aClassに対するクラス記述、またはあるクラス記述が見つけられなかったならばnil。

Discussion 議論

If a class description for aClass is not found, the method posts an NSClassDescriptionNeededForClassNotification on behalf of aClass, allowing an observer to register a class description. The method then checks for a class description again. Returns nil if a class description is still not found. aClassに対するあるクラス記述が見つけられないならば、このメソッドはNSClassDescriptionNeededForClassNotificationをaClassに代わって投函して、オブザーバがクラス記述を登録できるようにします。メソッドはそれからクラス記述を再び確認します。クラス記述が依然として見つけられないならば、nilを返します。

If you have an instance of the receiver’s class, you can use the NSObject instance method classDescription instead. あなたがレシーバの持つクラスのインスタンスを持つならば、あなたはNSObjectのインスタンスメソッドclassDescriptionを代わりに使用できます。

Note 注意

In macOS 10.6 and later, this method (and as a result classDescription methods of any object) will return nil when the sdef contains no <class> element for the Cocoa class, but there is a <class> element defined for a superclass. macOS 10.6以降では、このメソッド（そしてその結果あらゆるオブジェクトのclassDescriptionメソッド）はnilをsdefが<class>要素をCocoaクラスに対して含まない、しかしスーパークラスに対して定義される<class>要素が存在する場合には返します。

This is incorrect, as object instances should never be required to be exactly a given class, any class should be allowed to be a subclass of the required class and receive the correct <class> value. これは正しくありません、オブジェクトインスタンスは正確に与えられたクラスであることを要求されないはずなので、どんなクラスも要求されるクラスのサブクラスであることを許可されて正確な<class>値を受け取るはずです。

This situation can have a serious impact on Cocoa Scripting, and there is no plan on changing this behavior. この状況は、重大な影響をCocoaスクリプティング上で持つ可能性があります、そしてこの挙動を変更する計画はありません。

Instead of using this method, you should use the classDescriptionForClass: method of NSScriptClassDescription instead.

See Also 参照

Working with class descriptions クラス記述を扱う

+ invalidateClassDescriptionCache

Removes all NSClassDescription objects from the cache. 全てのNSClassDescriptionオブジェクトをキャッシュから除去します。

+ registerClassDescription:forClass:

Registers an NSClassDescription object for a given class in the NSClassDescription cache. 与えられたクラスに対するNSClassDescriptionオブジェクトをNSClassDescriptionキャッシュに登録します。
