Optimizing App Data for iCloud Backup

Minimize the space and the amount of time that iCloud Backup requires for your app data.

Framework フレームワーク

  • Foundation

On This Page この項には

  • Overview 
  • See Also  参照 

Overview 概要

iCloud Backup can automatically back up a user’s device. You can shorten the time it takes to back up your app, and minimize the space necessary for the backup in two ways:

  • Store your app’s data in the appropriate standard app directories

  • Mark files that iCloud doesn’t need to back up

Create a File in a Standard App Directory

iCloud Backup backs up standard app directories that contain user documents and app data, and ignores others that contain cached or temporary files.

To back up your app’s files, create them in the directories that iCloud Backup copies from the device:

  • User data files, located in <AppHome>/Documents/

  • Support files for the app, located in <AppHome>/Library/Application Support/

  • Other app data, such as private files, located in <AppHome>/Library/<app created directory>/

For items that don’t require backing up, create temporary files in the directories iCloud Backup doesn’t copy:

  • Data that your app can regenerate or download again, located in <AppHome>/Library/Caches/

  • Temporary data that the system can remove at any time, located in <AppHome>/tmp/. Note that the base path for the tmp directory may be different in iOS and macOS.

Use url(for:in:appropriateFor:create:) to find the path for the document, library, app support, and cache directories that you want to back up. Use temporaryDirectory to find the path for temporary data.

The code below shows how to create a file in any of the standard app directories. Create a file by calling createFile(filename:contents:attributes:) for the desired directory. For example, CreateFileDirectory.cache.createFile(”myCacheFile”, contents:nil, attributes:nil) creates a file called myCacheFile in the <AppHome>/Cache directory.

To find the base URL for a standard directory, use directoryURL. For example, CreateFileDirectory.documents.directoryURL returns the file URL for the <AppHome>/Documents directory.

Prevent iCloud Backup of a File

A file in a standard directory may not need saving by iCloud Backup, such as an automatic backup for an edited document. To prevent backup, set isExcludedFromBackupKey for the file. Set the key each time you save the file because certain common file operations clear the key.

To prevent backup of multiple files, create a subdirectory, set the directory’s isExcludedFromBackupKey, and move the files into that subdirectory.

In the code below, getExcludedFromiCloudBackup(fileURL) returns true if iCloud Backup excludes the file or directory. Set the value of isExcludedFromBackupKey for a file or directory using setExcludeFromiCloudBackup(fileURL:isExcluded:).
