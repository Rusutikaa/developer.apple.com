Increasing App Usage with Suggestions Based on User Activities ユーザアクションに基づく提案でアプリ利用を拡大する

Provide a continuous user experience by capturing information from your app and displaying this information as proactive suggestions across the system. 情報をあなたのアプリからキャプチャして、この情報を積極的提案としてシステムの至る所で表示することによって、途切れのないユーザ体験を提供してください。

Overview 概要

This sample app shows a list of pizza restaurants, near your current location, in a table view. Select a restaurant from the table view to donate its location information to the system as a user activity object. The system then displays the registered location information proactively as suggestions, such as in Search, used with Siri Shortcuts. この見本アプリは、あなたの現在の場所近くの、ピザレストランの一覧をテーブルビューで示します。レストランをテーブルビューから選んで、それの位置情報をシステムへとユーザアクティビティオブジェクトとして提供してください。システムはそれから登録された位置情報を積極的に提案として表示します、たとえば「Siriショートカット」で使われる、「検索」において。

When a user interacts with a proactive suggestion, the system continues the activity by associating the object’s activity type with the sample app. ユーザが積極的提案と相互作用する時、システムは、オブジェクトのもつアクティビティ型を見本アプリと結びつけることによって、アクティビティを継続します。

When donating location information, this sample app also verifies whether the previously captured information appears, as QuickType keyboard suggestions, within other apps installed on a device. 位置情報を提供する時、この見本アプリはまた、QuickTypeキーボード提案として、デバイス上にインストールされた他のアプリ内で、以前にキャプチャされた情報が現れるかどうか検証します。

Declare a User Activity Type ユーザアクティビティ型を宣言する

You implement proactive suggestions by determining specific activities that a user can perform in your app, and whose state you can recreate at a later time. The sample app has one user activity, view-location, which represents the user viewing details of a specific restaurant location. あなたは積極的提案の実装を、ユーザがあなたのアプリで実行できる特定のアクティビティそれら、そしてあなたが後で再度作成できるそれの状態を決定することによって行います。見本アプリは１つのユーザアクティビティ、view-locationを持ちます、それはユーザが眺めているある特定のレストランの場所の詳細を表します。

In the sample app, the target includes the view-location activity in its Information Property List file, as an entry with the key name . The type of this entry is Array, and each member is a String representing a supported user activity in reverse DNS notation. 見本アプリにおいて、ターゲットはview-locationアクティビティをそれのInformation Property Listファイルの中に、キー名をもつある登録項目として含みます。この登録項目の型はArrayです、そして各メンバーは、サポートされるユーザアクティビティを逆DNS表記で表しているStringです。

Manage a User Activity ユーザアクティビティを管理する

At runtime, you represent a user activity with the NSUserActivity object. You initialize a user activity object with a string identifier, the same one declared earlier in the Info.plist file. 実行時に、あなたはユーザアクティビティをNSUserActivityオブジェクトで表します。あなたは、ユーザアクティビティオブジェクトを、以前にInfo.plistファイルにおいて宣言されたものと同じ文字列識別子で初期化します。

In the sample app, LocationViewController manages a single NSUserActivity object, representing the view-location activity type. When you select a restaurant, the view controller sets its userActivity property to the view-location activity—registering the user activity object as the current activity, replacing any other activity previously sent to Siri or the system. 見本アプリにおいて、LocationViewControllerは、view-locationアクティビティ型を表している、ある単一のNSUserActivityオブジェクトを管理します。あなたがあるレストランを選ぶ時、ビューコントローラはそれのuserActivityプロパティをview-locationアクティビティに設定します — ユーザアクティビティオブジェクトを現在のアクティビティとして登録して、以前にSiriまたはシステムに送られた何らかの他のアクティビティを置き換えて。

Important 重要

When the user activity object is not contained within the responder chain, you must call the becomeCurrent() method to mark the object as current, which also registers the object with the system. ユーザアクティビティオブジェクトがレスポンダチェーン内に含まれない場合、あなたはbecomeCurrent()メソッドを呼び出すことで、オブジェクトを現在のものとして印しなければなりません、それはまたオブジェクトをシステムに登録します。

The view controller also sets needsSave to true, indicating that the activity will be updated with new data in the future, which eventually results in a callback to the updateUserActivityState(_:) method. This is the app’s opportunity to refresh the activity object’s userInfo property, with the minimal amount of information needed to restore the state of the app, before Siri or the system receives the activity. ビューコントローラはまた、needsSaveをtrueに設定します、そのアクティビティが新しいデータで将来に更新されることを指し示しています、それは結局はupdateUserActivityState(_:)メソッドへのコールバックという結果になります。これは、アプリにとって、アプリの状態を復元するのに必要とされる最小限の情報量で、Siriまたはシステムがアクティビティを受け取る前に、アクティビティオブジェクトのもつuserInfoプロパティをリフレッシュする好機です。

Provide Support for Improved Search Results 改善された検索結果に対するサポートを提供する

If your user activity objects contain information that the user might want to search for later, you can have Search index those objects and consider them during subsequent on-device searches. あなたのユーザアクティビティオブジェクトそれらが、ユーザが後で検索したいかもしれない情報を含むならば、あなたは「検索」にそれらオブジェクトのインデックスを作らせて、そして以降のデバイス上の検索の間にそれらを考慮に入れることができます、

To enable support for Search on a user activity object, set the isEligibleForSearch property to true. Additionally, you can make an activity object publicly accessible to all iOS users by setting the isEligibleForPublicIndexing property to true. ユーザアクティビティオブジェクト上で「検索」に対するサポートを可能にするには、isEligibleForSearchプロパティをtrueに設定してください。加えて、あなたはあるアクティビティオブジェクトを全てのiOSユーザに公然とアクセス可能にできます、isEligibleForPublicIndexingプロパティをtrueに設定することによって。

Provide the activity object with as much rich information about the activity as possible, by configuring the contentAttributeSet, keywords, or webpageURL properties so that the system can index the object. The app must also maintain a strong reference to any activity objects that the system uses for search results. アクティビティオブジェクトを、できるだけ豊かなアクティビティについての情報とともに提供してください、contentAttributeSet、keywords、またはwebpageURLプロパティを構成設定することによって、それでシステムはオブジェクトのインデックスをつくれます。アプリはまた、あらゆるアクティビティオブジェクトに対する強い参照を保守しなければなりません、それらはシステムが結果を検索するために使うものです。

To learn more about Search, see App Search Programming Guide. 「検索」についてもっと学ぶには、App Search Programming Guideを見てください。

Provide Support for Shortcuts ショートカットに対するサポートを提供する

Siri learns about the shortcuts available to your app through donations that your app makes to Siri. You can make donations from a user activity when the action involves a view within your app, such as the sample app, which displays a list of nearby restaurants. Siriは、あなたのアプリに利用可能なショートカットを、あなたのアプリがSiriにする提供を通して学びます。あなたは、アクションがビューをあなたのアプリ内、たとえば近くのレストランの一覧を表示する見本アプリなど、から発動する時に、ユーザアクティビティから提供をすることができます。

The sample app enables support for Siri Shortcuts on a user activity object by setting the isEligibleForPrediction property to true. The app suggests an invocation phrase, which is displayed to the user when they create a shortcut, by setting a short, memorable phrase to the suggestedInvocationPhrase property. 見本アプリは、isEligibleForPredictionプロパティをtrueに設定することによって、ユーザアクティビティオブジェクト上でSiriショートカットに対するサポートを可能にします。アプリは、ユーザがショートカットを作成する時にかれらに表示される発動フレーズを提案します、ある短い、覚えやすいフレーズをsuggestedInvocationPhraseプロパティに対して設定することによって。

During development, the system provides developer settings to modify the display behavior of shortcuts, and force sync shortcuts to an Apple Watch from an iPhone device. Go to Settings > Developer on the iPhone device to test an app’s behavior with Siri Shortcuts. 開発の間に、システムは、ショートカットの表示挙動を修正する、そして同期ショートカットをiPhoneデバイスからApple Watchへと強制するために、開発者設定を提供します。Settings > Developerに行って、アプリの挙動を「Siriショートカット」でテストしてください。

For more information, see Donating Shortcuts. さらなる情報は、Donating Shortcutsを見てください。

Handle Activities アクティティビティを取り扱う

When a user interacts with a proactive suggestion for your app, such as through Siri Shortcuts, the system restores your app to the foreground, receives data associated with its user activity, and calls application(_:continue:restorationHandler:). ユーザがあなたのアプリに対する積極的提案と相互作用を、たとえば「Siriショートカット」を通して行う時、システムはあなたのアプリを前面に戻して、そのユーザアクティビティと結びつけられたデータを受け取ります、そしてapplication(_:continue:restorationHandler:)を呼び出します。

To give SearchViewController a chance to perform its state restoration operation through restoreUserActivityState(_:), the sample app verifies the activity type of the user activity object, and passes the first tab’s view controller hierarchy to the restorationHandler block parameter. SearchViewControllerに、それの状態復元操作をrestoreUserActivityState(_:)を通して実行する機会を与えるために、見本アプリはユーザアクティビティオブジェクトのアクティビティ型を検証します、そして最初のタブのもつビューコントローラ階層をrestorationHandlerブロックパラメータに渡します。

The sample app validates the user activity, consumes any necessary information from the userInfo dictionary, and updates its UI to continue the requested activity from where the user left off. 見本アプリはユーザアクティビティを検証します、あらゆる必要な情報をuserInfoから使い果たします、そして要請されたアクティビティをユーザがやめたところから継続するために、それのUIを更新します。

Verify QuickType Keyboard Integration QuickTypeキーボード統合を検証する

Text fields describe the intended purpose of input data, like country name or postal code, via the textContentType property. If any app on the device recently donated information matching the expected text content type, that information is suggested in the QuickType keyboard as the user types in the text field. テキストフィールドは、入力データの意図される目的を、国名または郵便番号のように、textContentTypeプロパティによって、記述します。デバイス上のなんらかのアプリがその期待されるテキスト内容型と一致する情報を提供したならば、その情報はQuickTypeキーボードにおいてユーザがテキスト欄にタイプするにつれて提案されます。

For example, in the sample app, a user selects a pizza restaurant, and its address is suggested in Messages or a search field in Apple Maps. たとえば、見本アプリにおいて、ユーザはあるピザレストランを選択します、そしてそれのアドレスはMessagesまたはApple Mapsでの検索欄において提案されます。

To verify this behavior, select a point of interest in Apple Maps, then return to this sample app to see the donated information populated in the QuickType keyboard. この挙動を検証するには、Apple Mapsにおいて興味を引く地点を選択します、それからこの見本アプリに戻って、QuickTypeキーボードに現れる提供された情報を見てください。
