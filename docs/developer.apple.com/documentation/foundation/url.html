URL

A value that identifies the location of a resource, such as an item on a remote server or the path to a local file. あるリソース、例えばリモートサーバー上の項目の位置を識別する値、またはローカルファイルへのパス。

Declaration 宣言

Overview 概要

You can construct URLs and access their parts. For URLs that represent local files, you can also manipulate properties of those files directly, such as changing the file’s last modification date. Finally, you can pass URLs to other APIs to retrieve the contents of those URLs. For example, you can use the URLSession classes to access the contents of remote resources, as described in URL Session Programming Guide. あなたは、URLを組み立てるそしてそれの部分にアクセスすることができます。ローカルファイルを表すURLに対しては、あなたはまたそれらのファイルのプロパティを直接に操作できます、例えばファイルの持つ最終修正日を変更するなど。最終的に、あなたはURLを他のAPIに渡して、それらURLの内容を回収できます。例えば、あなたはURLSessionを使って、リモートリソースの内容にアクセスできます、URL Session Programming Guideで記述されるように。

URLs are the preferred way to refer to local files. Most objects that read data from or write data to a file have methods that accept a URL instead of a pathname as the file reference. For example, you can get the contents of a local file URL as String by calling func init(contentsOf:encoding) throws, or as a Data by calling func init(contentsOf:options) throws. URLは、ローカルファイルを参照する好ましい方法です。ファイルからデータを読み出すまたはそれへとデータを書き込むほとんどのオブジェクトは、URLをパス名の代わりにファイル参照として受け入れるメソッドを持ちます。例えば、あなたはローカルファイルURLの内容をfunc init(contentsOf:encoding) throwsを呼び出すことによってStringとして、func init(contentsOf:options) throwsを呼び出してDataとして取得できます。
