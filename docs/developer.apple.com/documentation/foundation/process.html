Process

An object representing a subprocess of the current process. 現在のプロセスの下位プロセスを表しているオブジェクト。

Declaration 宣言

Overview 概要

Using the Process class, your program can run another program as a subprocess and can monitor that program’s execution. A Process object creates a separate executable entity; it differs from Thread in that it does not share memory space with the process that creates it. Processクラスを使用して、あなたのプログラムは別のプログラムを下位プロセスとして実行できます、そしてそのプログラムの実行を監視できます。Processオブジェクトはある独立した実行可能形式存在を作成します；それはThreadとは違います、それにおいてそれはメモリ空間をそれを作成するプロセスと共有しません。

A process operates within an environment defined by the current values for several items: the current directory, standard input, standard output, standard error, and the values of any environment variables. By default, an Process object inherits its environment from the process that launches it. If there are any values that should be different for the process, for example, if the current directory should change, you must change the value before you launch it. A process's environment cannot be changed while it is running. プロセスは、いくつかの項目：現在のディレクトリ、標準入力、標準出力、標準エラー、そして何らかの環境変数の値に対して、現在の値によって定義されるある環境内で演算します。省略時では、Processオブジェクトは、それの環境をそれを起動するプロセスから継承します。異なるべき何らかの値がそのプロセスに対してあるならば、例えば、現在のディレクトリが変わるべきならば、あなたはその値をあなたがそれを起動する前に変更しなければなりません。プロセスのもつ環境は、それが実行中に変更されることはできません。

An Process object can only be run once. Subsequent attempts raise an error. Processオブジェクトは、一度だけ実行されることのみが可能です。その後の試みはエラーを引き起こします。

Important 重要

In a sandboxed application, child processes created with the Process class inherit the sandbox of the parent app. You should generally write helper applications as XPC Services instead, because XPC Services allows you to specify different sandbox entitlements for helper apps. See Daemons and Services Programming Guide and XPC for more information. サンドボックス化されたアプリケーションでは、Processクラスで作成される子プロセスは、親アプリのサンドボックスを継承します。あなたは、一般にヘルパーアプリケーションをXPC Serviceとして代わりに書くべきです、なぜならXPC Serviceはあなたに異なるサンドボックス資格をヘルパーアプリケーションそれらに指定できるようにするからです。Daemons and Services Programming GuideとXPCをさらなる情報として見てください。

Running and Stopping a Task タスクの実行と停止

func interrupt()

Sends an interrupt signal to the receiver and all of its subtasks. 割り込み信号をレシーバとそれの下位タスクの全てに送ります。

func launch()

Launches the task represented by the receiver. レシーバによって表されるタスクを起動します。

Deprecated 非推奨
func resume() -> Bool

Resumes execution of the receiver task that had previously been suspended with a suspend() message. 以前にsuspend()メッセージで一時停止されたレシーバタスクの遂行を再開します。

func suspend() -> Bool

Suspends execution of the receiver task. レシーバタスクの遂行を一時停止します。

func terminate()

Sends a terminate signal to the receiver and all of its subtasks. 終止信号をレシーバとそれの下位タスクの全てに送ります。

func waitUntilExit()

Block until the receiver is finished. レシーバが終了されるまで遮断します。

Querying the Task State タスク状態を問い合わせる

var isRunning: Bool

Returns whether the receiver is still running. レシーバがまだ実行中かどうかを返します。

var terminationStatus: Int32

Returns the exit status returned by the receiver’s executable. レシーバのもつ実行可能ファイルによって返される退出状態を返します。

var terminationReason: Process.TerminationReason

Returns the reason the task was terminated. タスクが終わらされた理由を返します。

Constants 定数

enum Process.TerminationReason

These constants specify the values that are returned by terminationReason. これら定数は、terminationReasonによって返される値を指定します。

enum QualityOfService

Used to indicate the nature and importance of work to the system. Work with higher quality of service classes receive more resources than work with lower quality of service classes whenever there is resource contention. 仕事の性質と重要性をシステムに指し示すために使われます。より高い質のサービスクラスを扱う作業は、より低い質のサービスクラスを扱う作業より多くのリソースを受け取ります、リソース競合がある時はいつでも。

Notifications 通知

class let didTerminateNotification: NSNotification.Name

Posted when the task has stopped execution. This notification can be posted either when the task has exited normally or as a result of terminate() being sent to the NSTask object. If the NSTask object gets released, however, this notification will not get sent, as the port the message would have been sent on was released as part of the task release. The observer method can use terminationStatus to determine why the task died. See Ending an NSTask for an example. タスクが遂行を停止した時に投函されます。この通知は、タスクが通常通りに退出した時またはNSTaskオブジェクトに送られるterminate()の結果としてのどちらでも投函されることができます。NSTaskオブジェクトが解放されたならば、しかしながら、この通知は送られないでしょう、メッセージが送られたかもしれないポートが、タスク解放の一部として解放されたので。オブザーバメソッドは、terminationStatusを使ってなぜタスクが死んだのか決定できます。Ending an NSTask を例として見てください。
