Identifying People, Places, and Organizations 人々、場所、および組織を識別する

Use a linguistic tagger to perform named entity recognition on a string. 言語学的タガーを使って、ある文字列上で固有表現抽出を実行します。

Framework フレームワーク

  • Foundation

On This Page この項には

  • Overview 
  • See Also 

Overview 概要

Identifying named entities in natural language text can help make your app more intelligent. For example, a messaging app might look for names of people and places in text in order to display related information like contact information or directions. 自然言語テキストにおいて固有表現を識別することは、あなたのアプリをより知的なものにする助けとなりえます。例えば、メッセージングアプリは人々や場所の名前をテキストにおいて捜すかもしれません、問い合わせ先や道順のような関連情報を表示する目的で。

The example below shows how to use NSLinguisticTagger to enumerate over natural language text and identify any named person, place, or organization. 下の例は、NSLinguisticTaggerを使って、自然言語テキストを列挙して、何らかの指定の個人、場所、または組織を識別する方法を示します。。

First, an instance of NSLinguisticTagger is created, specifying nameType as the tag scheme to be used. Next, the string property of the linguistic tagger is set to the natural language text. Finally, the linguistic tagger enumerates over the entire range of the string, specifying NSLinguisticTaggerUnit.word as the tag unit and nameType as the tag scheme, omitting any punctuation or whitespace, and joining words that are part of a single name into the same token. In the enumeration block, the name type is provided by tag, and each word is obtained by taking a substring of the original text at tokenRange. 最初に、NSLinguisticTaggerのインスタンスが作成されて、nameTypeを使用タグスキームとして指定します。次に、その言語学的タガーのstringプロパティが自然言語テキストに設定されます。最後に、言語学的タガーは文字列の範囲全体を列挙します、NSLinguisticTaggerUnit.wordをタグ単位としてそして nameTypeをタグスキームとして指定して、そしてあらゆる句読点や空白を省略して、そして単一の名前の一部である単語それらを同じトークンへと結合して行います。列挙ブロックにおいて、名前の型はtagによって提供されます、そして各単語は元のテキストの下位文字列をtokenRangeで取得することで得られます。
