Initializer

init(contentsOf:options:)

Initializes a data object with the data from the location specified by a given URL. あるデータオブジェクトをある与えられたURLによって指定される場所からのデータで初期化します。

SDKs

  • iOS 2.0+
  • macOS 10.4+
  • Mac Catalyst 13.0+
  • tvOS 9.0+
  • watchOS 2.0+

Framework フレームワーク

  • Foundation

Declaration 宣言

Parameters パラメータ

aURL

The URL from which to read data. それからデータを読み出すことになるURL。

mask

A mask that specifies options for reading the data. Constant components are described in NSData.ReadingOptions. データ読み出しに対するオプションを指定するあるマスク。定数構成要素それらは、NSData.ReadingOptionsにおいて記述されます。

Discussion 議論

Use this method to convert data:// URLs to NSData objects. You can also use it to read short files synchronously. このメソッドを使ってdata://URLをNSDataオブジェクトに変換してください。あなたはまたそれを使って短いファイルそれらを同期的に読み出せます。 If you need to read potentially large files, initialize an InputStream with inputStreamWithURL:, then read the file incrementally.

Important 重要

Don't use this synchronous initializer to request network-based URLs. For network-based URLs, this method can block the current thread for tens of seconds on a slow network, resulting in a poor user experience, and in iOS, may cause your app to be terminated. ネットワーク基盤のURLに対して、このメソッドは低速なネットワーク上で10秒の間現在のスレッドをブロックして、劣悪なユーザ体験という結果になる可能性があります、そしてiOSでは、あなたのアプリは終了させられるかもしれません。

Instead, for non-file URLs, consider using the dataTask(with:completionHandler:) method of the URLSession class. See Fetching Website Data into Memory for an example. 代わりに、非ファイルURLに対して、URLSessionクラスのdataTask(with:completionHandler:)メソッドの使用を考えてください。Fetching Website Data into Memoryをひとつの例として見てください。

Handling Errors in Swift: Swiftでのエラー処理：

In Swift, this API is imported as an initializer and is marked with the throws keyword to indicate that it throws an error in cases of failure. Swiftでは、このAPIはイニシャライザとしてインポートされてthrowsキーワードで印されることで、それが様々な失敗においてエラーをスローすることを指し示します。

You call this method in a try expression and handle any errors in the catch clauses of a do statement, as described in Error Handling in The Swift Programming Language and About Imported Cocoa Error Parameters. あなたはこのメソッドをtry式の中で呼び出して、あらゆるエラーをdo文のcatch節で取り扱います、The Swift Programming Languageのエラー処理そしてインポートされるCocoaエラーパラメータについてで記述されるように。
