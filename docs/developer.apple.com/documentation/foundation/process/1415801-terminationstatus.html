Declaration 宣言

Return Value 戻り値

The exit status returned by the receiver’s executable. レシーバのもつ実行可能ファイルによって返される退出状態。

Discussion 議論

Each task defines and documents how its return value should be interpreted. For example, many commands return 0 if they complete successfully or an error code if they don’t. You’ll need to look at the documentation for that task to learn what values it returns under what circumstances. 各タスクは、どのようのそれの戻り値が解釈されるべきかを定義および文書化します。例えば、多くのコマンドは、もしそれらが成功裏に完了するならば0をまたはそれらがそうしないならばエラーコードを返します。あなたは、そのタスクに対する文書化を捜してどんな値をそれは返すのかどんな事情でかを学ぶ必要があるでしょう。

This method raises an NSInvalidArgumentException if the receiver is still running. Verify that the receiver is not running before you use it. このメソッドは、NSInvalidArgumentExceptionを、もしレシーバがまだ実行中ならば返します。レシーバが実行中でないことをあなたがそれを使う前に検証してください。

See Also 参照

Querying the Task State タスク状態を問い合わせる

var isRunning: Bool

Returns whether the receiver is still running. レシーバがまだ実行中かどうかを返します。

var terminationReason: Process.TerminationReason

Returns the reason the task was terminated. タスクが終わらされた理由を返します。
