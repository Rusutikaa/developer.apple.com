Protocol

View

A type that represents part of your app’s user interface and provides modifiers that you use to configure views. ある型、それはあなたのアプリのもつユーザインターフェイスの一部を表します、そしてあなたがビューを構成設定するのに使う修飾子を提供します。

Declaration 宣言

Overview 概要

You create custom views by declaring types that conform to the View protocol. Implement the required body computed property to provide the content for your custom view. あなたは、あつらえのビューをViewプロトコルに準拠する型を宣言することによって作成します。必須のbody計算プロパティを実装することで、あなたのあつらえのビューに対する内容を提供してください。

Assemble the view’s body by combining one or more of the primitive views provided by SwiftUI, like the Text instance in the example above, plus other custom views that you define, into a hierarchy of views. ビューのもつボディの組み立てを、SwiftUIによって提供される１つ以上のプリミティブビューを、上の例でのTextインスタンスのように、それに加えてあなたが定義する他のあつらえのビューを、ビューそれらからなるある階層へと結合することによって行ってください。

The View protocol provides a large set of modifiers, defined as protocol methods with default implementations, that you use to position and configure views in the layout of your app. Modifiers typically work by wrapping the view instance on which you call them in another view with the specified characteristics. For example, adding the opacity(_:) modifier to a text view returns a new view with some amount of transparency: Viewプロトコルは、省略時の実装を備えるプロトコルメソッドとして定義される、広範なひとそろいの修飾子を提供します、それはあなたが使用してあなたのアプリのレイアウトにおいてビューの位置を定めたり構成設定をするものです。それら修飾子は、概してビューインスタンスをラップすることによって働きます、それの上であなたはその指定された特徴をもつ別のビューにおいてそれらを呼び出します。例えば、opacity(_:)修飾子をテキストビューに加えることは、ある量の透明度での新しいビューを返します：

The effects of a modifier typically propagate to any child views that don’t explicitly override the modifier. For example, a VStack instance on its own acts only to vertically stack other views, and has no text to display. Therefore, a font(_:) modifier that you apply to the stack has no effect on the stack itself. Yet the font modifier does apply to any of the stack’s child views, some of which might display text. On the other hand, you can locally override the stack’s modifier by adding another to a specific child view: ある修飾子の効果は概して、その修飾子を明示的にオーバーライドしないあらゆる子ビューに伝播します。例えば、VStackインスタンスは独立して、垂直に他のビューを積みかさるためだけの役割をします、そして表示するテキストを持ちません。それゆえ、あなたがスタックに適用するfont(_:)修飾子は、スタックそれ自身に影響を及ぼしません。それでもフォント修飾子は、スタックのもつ子ビューのどれに対しても適用されます、それのいくつかはテキストを表示するかもしれません。他方では、あなたはローカルにスタックのもつ修飾子をオーバーライドすることが、別のものを特定の子ビューに加えることによって可能です：

You commonly chain modifiers, each wrapping the result of the previous one. For example, you can wrap a text view in an invisible box with a given width using the frame(width:height:alignment:) modifier to influence its layout, and then use the border(_:width:) modifier to draw an outline around that: あなたは普通に修飾子を連鎖状にします、それぞれ前のものの結果をラップしています。例えば、あなたは可視ボックスの中のテキストビューをframe(width:height:alignment:)修飾子を使ってラップすることでそれのレイアウトに影響を与えて、それからborder(_:width:)修飾子を使ってそれの周囲に輪郭を描画できます：

The order in which you apply modifiers matters. For example, the border that results from the above code outlines the full width of the frame. あなたが修飾子を適用する順番は重要です。例えば、上のコードの結果である境界線は、そのフレームの幅いっぱいに輪郭を描きます。

If you instead apply the border first, it outlines the text view, which never takes more space than it needs to render its contents. あなたが代わりに境界線を最初に適用するならば、それはテキストビューの輪郭を描きます、それは決してそれがそれの内容を描出するのに必要とするより多くの空間を取りません。

Wrapping that view in another invisible one with a fixed 100 point width affects the layout of the composite view, but has no effect on the border. そのビューを別の不可視のものの中に固定された100ポイント幅でラップすることは、合成ビューのレイアウトに影響します、しかし境界線に効果を及ぼしません。

Topics 話題

Implementing a Custom View カスタムビューを実装する

var body: Self.Body

The content and behavior of the view. ビューの内容と挙動。

Required. Default implementations provided. 必須。 提供される省略時の実装いくつか。

Configuring Views ビューを構成設定する

Layout レイアウト

Tell a view how to arrange itself within a view hierarchy by adjusting its size, position, alignment, padding, and so on. ビューにどのようにそれ自身をビュー階層内で整えるか伝えます、それの大きさ、位置、整列、詰め、その他を調整することによって。

Accessibility アクセシビリティ

Make your views accessible to a broad range of users, including people with disabilities. あなたのビューを広範囲のユーザにアクセス可能にします、身体障害をもつ人々を含めて。

Rendering レンダリング

Affect the way the system draws a view, for example by scaling or masking a view, or by applying graphical effects. システムがビューを描画する方法に影響を与えます、例えばあるビューを拡大縮小またはマスクすることによって、またはグラフィカル効果を適用することによって。

Styling スタイリング

Indicate how to style the text, controls, and other content contained in a view. どのようにビューの中に含まれるテキスト、コントロール、および他の内容をスタイル指定するかを指し示します。

Providing Interactivity 相互作用性を提供する

Input and Events 入力とイベント

Supply actions for a view to perform in response to user input and system events. アクションをビューに提供して、ユーザ入力とシステムイベントに応答して実行します。

View Presentation ビュー提示

Define additional views for the view to present under specified conditions. 追加のビューをビューに対して定義して、指定された条件の下で提示します。

State 状態

Access view preferences and provide child views with configuration data. ビュー環境設定にアクセスします、そして子ビューを構成設定されたデータで提供します。

Configuring Previews in Xcode Xcodeにおいてプレビューを構成設定する

func previewDevice(PreviewDevice?) -> View

Overrides the device for a preview. プレビューに対するデバイスをオーバーライドします。

func previewDisplayName(String?) -> View

Provides a user visible name shown in the editor. エディタにおいて示されるユーザ可視の名前を提供します。

func previewLayout(PreviewLayout) -> View

Overrides the size of the container for the preview. プレビューに対するコンテナの大きさをオーバーライドします。

Identifying Views ビューを識別する

func id<ID>(ID) -> View

Generates a uniquely identified view that can be inserted or removed. 特有に識別されるビューを生成します、それは挿入または除去されることができます。

func equatable() -> EquatableView<Self>

Prevents the view from updating its child view when its new value is the same as its old value. ビューがそれの子ビューを更新することを、それの新しい値がそれの古い値と同じ場合に防止します。

Available when Self conforms to Equatable. SelfがEquatableに準拠する場合に利用可能です。
associatedtype Body

The type of view representing the body of the view. ビューのボディを表しているビューの型。

Required. 必須。

Implementing View Modifiers ビュー修飾子を実装する

func modifier<T>(T) -> ModifiedContent<Self, T>

Applies a modifier to a view. 修飾子をビューに適用します。

struct ModifiedContent

A value with a modifier applied to it. それに適用された修飾子をもつある値。

struct EmptyModifier

The empty, or identity, modifier. 空の、または同一性、修飾子。
